# Copywriting Skill

Write and review copy for Oblique clients following the Oblique Copywriting Framework.

## Who this is for

Sean Ng, CEO of Oblique — a branding and performance marketing agency in KL, Malaysia.
All copy must meet Oblique's standard before it reaches Sean. The framework is the ruleset.
Don't deliver copy that fails its own self-review.

## Workflow

### Step 1: Read the framework first

Before writing a single word, read the Copywriting Framework from the vault path above.
The rules evolve over time — always read the current version, don't rely on memory.

### Step 2: Gather context

Ask Sean three questions before writing. Use the AskUserQuestion tool:

1. **Format** — Landing page / Static ad (Meta/IG) / Product description / Social post / Proposal / Other
2. **Audience** — Who is this for? (pain point, lifestyle, demographics — whichever is most relevant)
3. **Angle** — What's the main thing this copy needs to make the reader feel or believe?

If Sean has already specified any of these, skip the relevant question.

### Step 3: Pull client context from the vault

Check if a client file exists at:
`/sessions/fervent-nice-gauss/mnt/Sean's Oblique Vault/10 - Oblique/Clients/[ClientName].md`

Also check for any related files (e.g. `Getha New Launch.md`, `Getha Ads Priority.md`).
Read whatever exists. Pull specific product claims, numbers, positioning notes, or brand direction.
Specific product facts in the copy are always better than generic category claims.

### Step 4: Write the copy

Apply the framework as you write — not as a post-process. The rules should shape the sentences
as they're written, not be applied as corrections afterwards.

Format-specific guidance:

**Static ads:** One punchy outcome line the reader can see themselves in. No follow-up needed.
The line must work in isolation with zero product context.

**Landing pages:** Hero triggers emotion/imagination. Subheadline earns the right to the product
story. Body sections lead with what the reader gets, then explain what makes it possible.
CTA is specific and low-friction.

**Product descriptions:** Specific claims over category claims. Lead with the experience,
follow with the spec. Never describe what a product "is" before you've described what
it does for the person buying it.

**Social posts:** Voice matches the brand (not Oblique's voice — the client's voice).
Lead with the hook. One idea per post. No hashtag stuffing.

### Step 5: Self-review before delivering

Read the self-review checklist:
`/sessions/fervent-nice-gauss/mnt/.claude/skills/copywriting/references/self-review.md`

Run every item on the list against the copy. Fix any failures before delivering.
This step is not optional — the point is that Sean receives clean copy, not a draft.

### Step 6: Deliver

Present the copy clearly. After the copy, include a short note (2–4 lines) covering:
- Any factual claims that need client verification before going live (numbers, guarantees, etc.)
- Any deliberate calls you made that Sean might want to adjust (tone, angle, CTA)
- Nothing else — no lengthy explanation, no self-congratulation

## Reviewing existing copy

When Sean asks you to review copy (rather than write it), read the framework first,
then give feedback in prose — direct, editorial, senior copywriter giving notes.
Not a checklist. Not a rubric. Specific line-level callouts: what's wrong, why in one
sentence, what to do instead. Follow the tone standard in Rule 7 of the framework.
